- **SQL Dialect**: SQLite (not PostgreSQL). Use SQLite-compatible syntax and functions.
  - For date operations use: `DATE()`, `DATE('now')`, `STRFTIME()` (not PostgreSQL `DATE_TRUNC()` or `INTERVAL`)
  - For date arithmetic use: `DATE('now', '+1 month')` or `DATE('2025-01-01', '+30 days')` (not PostgreSQL `INTERVAL`)
  - For string operations use SQLite functions: `UPPER()`, `LOWER()`, `SUBSTR()`, `INSTR()`

## Date Range Queries (Last N Days)

When filtering for "last N days", use the boundary formula: **N-1 intervals inclusive**.

**Why:** "Last 7 days" means the past 7 calendar days *including today*. Today is "0 days ago," so we count back 6 days. The boundary must be **inclusive** (>=) to include today.

**Current date: 2026-06-11**

### Worked Examples

- **Last 7 days** (2026-06-05 through 2026-06-11):
  ```sql
  WHERE posted_at >= date('now', '-6 days')
  ```
  Excludes 2026-06-04 and earlier.

- **Last 14 days** (2026-05-29 through 2026-06-11):
  ```sql
  WHERE posted_at >= date('now', '-13 days')
  ```
  Excludes 2026-05-28 and earlier.

- **Last 30 days** (2026-05-13 through 2026-06-11):
  ```sql
  WHERE posted_at >= date('now', '-29 days')
  ```
  Excludes 2026-05-12 and earlier.

**NOTE**: The boundary uses `>=` (inclusive). Always use `N-1` in the date modifier formula.
