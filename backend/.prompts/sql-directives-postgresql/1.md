- **SQL Dialect**: PostgreSQL (not SQLite). Use PostgreSQL-compatible syntax and functions.
  - For date operations use: `DATE_TRUNC()`, `NOW()`, `CURRENT_DATE`, `INTERVAL` (not SQLite `DATE()` or `STRFTIME()`)
  - For date arithmetic use: `NOW() - INTERVAL '1 month'` or `CURRENT_DATE + INTERVAL '30 days'` (not SQLite `DATE()` modifiers)
  - For string operations use PostgreSQL functions: `UPPER()`, `LOWER()`, `SUBSTRING()`, `POSITION()`

## Date Range Queries (Last N Days)

When filtering for "last N days", use the boundary formula: **N-1 intervals inclusive**.

**Why:** "Last 7 days" means the past 7 calendar days *including today*. Today is "0 days ago," so we count back 6 days. The boundary must be **inclusive** (>=) to include today.

**Current date: 2026-06-11**

### Worked Examples

- **Last 7 days** (2026-06-05 through 2026-06-11):
  ```sql
  WHERE posted_at >= CURRENT_DATE - INTERVAL '6 days'
  ```
  Excludes 2026-06-04 and earlier.

- **Last 14 days** (2026-05-29 through 2026-06-11):
  ```sql
  WHERE posted_at >= CURRENT_DATE - INTERVAL '13 days'
  ```
  Excludes 2026-05-28 and earlier.

- **Last 30 days** (2026-05-13 through 2026-06-11):
  ```sql
  WHERE posted_at >= CURRENT_DATE - INTERVAL '29 days'
  ```
  Excludes 2026-05-12 and earlier.

**NOTE**: The boundary uses `>=` (inclusive). Always use `N-1` in the INTERVAL formula.
