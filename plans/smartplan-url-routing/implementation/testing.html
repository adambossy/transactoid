Execution · deep dive

Testing

What the existing Playwright suite already demands, which specs the change touches, and the new coverage that proves the acceptance criteria.

Requirements

  1. Every acceptance criterion on the bead — routable views, deep-linkable conversations, working back/forward — is proven by an automated test, not a manual claim.
  2. The pre-existing test that forbids one user opening another user's conversation URL passes for real, instead of being skipped.

The suite today, and what the change touches

Good news first: no spec seeds or reads the local-storage session key, so retiring it breaks no fixtures. Navigation in the suite is plain goto() calls on the paths this plan keeps (/ ×9, /settings/providers, /invites, /ui, /sign-up), which continue to work unchanged.

One spec deserves special mention — it was written ahead of this feature and currently only runs Clerk-gated (PENNY_E2E_CLERK=1):

  • conversation-isolation.spec.ts already encodes this plan's contract: after user A sends a first message, pageA.url() is expected to be /c/<conversation-id>, and user B opening that URL must see a visible not found / no access message with zero assistant messages. Today this spec cannot pass (the URL never changes and a foreign id shows an empty chat). Stage 2 makes it honest — it is effectively the feature's pre-written acceptance test.
  • chat-smoke.spec.ts and the other /-based chat specs — behavior at / is still "empty chat, send a message, reply streams", so they should pass as-is; the only new observable is the URL changing after the first send, which does not invalidate their assertions.

New specs (stage 3)

A new routing.spec.ts covering, in dev-principal mode (so it runs in every suite invocation, not just Clerk-gated):

  1. URL replacement on first send — start at /, send a message, expect the URL to become /c/<id> without a page load (the transcript element identity survives).
  2. Deep-link hydration — capture that URL, goto() it in a fresh page, expect the same transcript to render.
  3. Back/forward — create conversation A, start a new chat, create conversation B, then walk goBack()/goForward() and assert the right transcript (and drawer highlight) at each step.
  4. Not-found state — goto("/c/" + randomUUID()), expect the "Conversation not found" message and no composer send into it.
  5. Unknown path — goto("/nonsense"), expect to land on /.

Manual checks

Things worth a human pass in the dev loop, because they involve feel or timing that specs approximate poorly:

  • Drawer switching between long conversations — no flash, header/drawer stay put, scroll lands at the bottom of the new transcript.
  • Refresh mid-stream on /c/<id> — the reply resumes (existing behavior, verify routing didn't disturb it).
  • Mobile-width drawer: tapping a conversation navigates and closes the overlay.
  • Cmd-click a drawer row — opens that conversation in a new tab (a new capability real links buy us).
  • In Clerk mode: sign out, deep-link a conversation, sign in — lands on that conversation.
