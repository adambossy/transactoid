Open questions

  • Should the drawer navigate-and-close also refresh its list? Today the list refetches on each open, which still holds. But a conversation started at / only appears in the list after the backend titles it — if that feels stale in practice, a refetch-on-navigate may be worth adding. Affects drawer links; decide from feel during the stage-2 manual pass.
  • Does /settings/providers want a /settings parent eventually? Out of scope here (one settings screen exists), but the route table should not preclude it. Affects the route contract only if more settings screens land.
  • Is the Clerk-gated isolation spec runnable in CI today? It requires PENNY_E2E_CLERK=1 plus Clerk test keys. If CI never sets that, the feature's strongest acceptance test only runs locally — worth confirming while wiring stage-3 coverage.
