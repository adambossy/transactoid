When things go wrong

Failure modes & ops

No-op on empty/incomplete; fail closed on branch/connection errors; perpetual retention is R2, branches transient.

Requirements

  1. A failed or incomplete run shows up as a clear gap, never as a falsely good or bad number.
  2. A misconfigured connection can never cause the evaluation to run against real production data.
  3. Unreviewed work is visible so branches don't quietly accumulate.

No-op cleanly (exit 0, record a marker)

  • Empty cohort (uncommon at 12h, ~10 txns/run): write eval_runs(status=skipped_empty), cut no branch.
  • Incomplete legacy baseline (this run's sync failed / hadn't run): write status=skipped_incomplete_baseline. Measuring the agent against a hole would corrupt the trend.

A skipped marker keeps the trend honest (a visible gap) instead of a missing or fabricated point.

Fail closed (exit non-zero, write nothing)

  • Branch creation fails (Neon quota/API): abort before any write.
  • Connection/host error: if the branch endpoint can't be resolved correctly, never fall back to prod — the connection-string --branch-name bug makes this a live footgun. Build from the branch's own endpoint host; if that fails, stop.

Pattern to mirror: the existing sync tool's structured catch-and-return-error, no partial writes.

Retention (R2) & orphan cleanup

Perpetual retention lives in R2, not Neon: at the end of each run the branch is dumped to a SQLite fixture in R2 and deleted. A year of 12-hour runs is a few GB of objects — pennies. Neon branches are created and deleted within a run, so they stay within the included allowance at ~$0, and there is no review backlog to worry about — a branch never outlives its run.

Orphan branches. A crash between create and delete can leave a branch behind. A reaper sweeps any eval/* branch whose run already has an r2_fixture_url (or is older than a run's max duration), so orphans can't accumulate.
Backtest fidelity. Backtests hydrate SQLite (dev-only in Penny) from the R2 fixture, so they may differ subtly from the Postgres run. Acceptable for a non-production backtest; the live measurement always runs on the Neon branch.
