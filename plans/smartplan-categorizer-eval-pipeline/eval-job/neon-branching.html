Branch + fixture

Branch lifecycle & R2 fixtures

A transient Neon branch for the live run; at the end of the run it becomes a cheap SQLite fixture in R2 and is deleted.

Requirements

  1. The live evaluation always connects to an isolated snapshot, never accidentally to production.
  2. Every eval is preserved forever cheaply, and can be re-run against a newer categorizer.
  3. Branches never accumulate cost, because each is deleted at the end of its run.

Creating the branch

Cut eval/<timestamp> off prod with neonctl branches create with a compute endpoint. Copy-on-write, so it is instant and cheap even off a large prod DB. The branch is transient — it exists only for the duration of the run, not for retention.

The connection-string footgun

Do not use neonctl connection-string --branch-name — a documented CLI bug returns the parent endpoint, so you would silently connect to prod. Build the URL from the branch's own endpoint host (see the backend/.env.test header, which already documents this for the test branch). Getting this wrong is exactly why the job must fail closed.

End of run: dump to R2 → delete

As the last step of the same unattended run — no human action, no waiting — the job retires the branch:

  1. Dump the branch's categorizer-reachable tables (~14 MB) into a fresh SQLite file by copying table-by-table through the shared SQLAlchemy models (not pg_dump — the models keep types and CHECK constraints dialect-correct).
  2. Upload the file (gzipped, ~3 MB) to R2 via Penny's storage adapter and record its URL as r2_fixture_url on the run.
  3. Delete the Neon branch.

The eval then lives in perpetuity as the prod eval_* rows plus the R2 fixture. A backtest of a future categorizer version downloads the fixture, hydrates a throwaway SQLite, and replays categorize_one against that exact frozen input + history.

Cost. Branches are created and deleted within a single run, so they stay within Neon's included allowance — effectively $0. Perpetual storage is R2: a few MB per run at ~$0.015/GB-month, so a year of runs is pennies. The $500–$3,000/month branch-retention cost of the earlier design is gone, and so is any review-backlog concern — a branch never outlives its run.
Caveat. SQLite is dev-only in Penny, so backtests carry a minor fidelity caveat vs. the Postgres run — acceptable for a non-production backtest. A reaper still sweeps orphan branches left by a crash between create and delete.
