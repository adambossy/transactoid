What you decided, the defaults I picked, and the questions still worth your attention.

What I asked you

QuestionYour answer
Grade prod's stored decision, or re-run the agent in eval? Re-run the agent and A/B it against the legacy decision; human adjudicates.
Keep the "blind" eval mechanism? No — rip it out entirely as part of this change.
Review/correction UX for the MVP? Chat-based via recategorize_transaction; dedicated HTML UI deferred.
Does Langfuse give us the verdict/dataset store? Investigated → no (OTEL push-only); Langfuse stays a debugging deep-link.
Sync & eval cadence? Every 12 hours (sync moves from hourly); the eval runs as part of each sync — ~10 transactions/run.
Retention mechanism & storage engine? Neon branch is transient; at the end of each run, dump to R2 as a SQLite fixture and delete the branch (no human gate). The eval lives in perpetuity as prod rows + an R2 reference; backtests hydrate SQLite.
Is SQLite first-class in Penny? No — Neon/Postgres is first-class; SQLite is dev-only and used here only to hydrate R2 fixtures for backtests (slow/quirky is acceptable there).
How many transactions per day/run? (measured live prod, 2026-06-21) ~20–23/day, ~10 per 12-hour run; clustered, so cohorts are rarely empty at this cadence.

Open questions & resolutions

The questions raised earlier, resolved on 2026-06-27 except where noted:

  • Auto-confirm fully-agreed cohorts? — YES (no stamp). There is no stamp step at all; runs where legacy and agent agree on every row simply send no email and need no review (still dumped to R2). Only disagreement runs are emailed. See Human review.
  • Notification channel — EMAIL. Each run that needs review is emailed to the owner. See Human review.
  • Legacy retirement — eval KEEPS RUNNING. It continues even after the agent replaces legacy in prod; it stays a permanent quality monitor (the baseline becomes the then-incumbent categorizer). See Goals.
  • Ship threshold numbers — DROPPED. No fixed "ready" state; the agent is improved gradually and the trend is a continuous signal, not a gate. See Goals.
  • Review backlog vs. branch lifetime — NOT A CONCERN. Deliberately ignored at current volume.
  • Confidence calibration — LATER. Plotting confidence-vs-correctness is worthwhile eventually; not in the initial build. Affects Metrics scope.
