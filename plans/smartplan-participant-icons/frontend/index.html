Frontend

New avatar primitives in the shared UI package, one members hook, and two surgical component changes — the sidebar entry and the chat message map. The shared @adambossy/agent-ui library is not modified.

Requirements

  1. Shared conversations in the history drawer show two small overlapping member pictures on the right of the entry, without crowding the title.
  2. In a shared conversation, the other member's messages are green, sit on the left, and carry their picture; the reader's own messages look exactly as they do today.
  3. Members without a picture still get a recognisable mark — their initial in a colored circle — so the UI never shows a broken image.

Data plumbing

One fetch feeds both features: GET /api/household/members is loaded once per app session by useHouseholdMembers and shared from AppShell down to the drawer and the chat screen — the same prop-threading pattern already used for getToken. From the response the client derives:

  • me — the member with is_you: true (the anchor for "mine vs theirs" in the transcript);
  • membersById — the avatar/name lookup for both the sidebar stack and the bubble icon;
  • members.length >= 2 — half of the sidebar's multi-user rule.

Per-message sender ids arrive separately on the session hydration payload (see Sender attribution) and are kept in a small sendersById map inside ChatScreen — deliberately not threaded through the AI SDK's message state, so no fights with useChat's typing or state management.

Why agent-ui stays untouched

The library's Message component takes only {message, isStreaming} — no author, no avatar, no variant. Teaching it about senders would push a Penny-specific concept (household members) into a general-purpose library whose message type has no sender field at all.

Instead, ChatScreen already owns the per-message wrapper <div data-role=…>, and the user-bubble markup is a single styled div — trivial to reproduce locally. The plan: render a small Penny-local OtherUserMessage for the one new case, delegate everything else (own messages, assistant messages, streaming, tool parts, reasoning) to the library exactly as today. If a second consumer of authored messages ever appears in agent-ui's world, that is the moment to generalize upstream — not before.
