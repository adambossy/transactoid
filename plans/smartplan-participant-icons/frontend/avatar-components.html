Avatar primitives

Two small components in the shared UI package — Avatar and AvatarStack — plus the useHouseholdMembers hook that feeds them. These are the only new reusable pieces in the project.

Requirements

  1. A member's picture renders as a clean circle at whatever small size the surrounding UI needs.
  2. A member without a picture shows their initial in a colored circle — never a broken-image glyph, never an empty hole.
  3. Two pictures can overlap into a compact stack that reads as "these people" at sidebar size.

Avatar

New primitive in frontend/packages/ui/src (exported from index.ts), beside IconButton/Chip. Props: { name: string; imageUrl?: string | null; size?: "xs" | "sm" | "md" } — sizes map to 20 / 24 / 32 px.

  • With imageUrl: <img> with rounded-full object-cover, alt={name}, and referrerPolicy="no-referrer" (Google-hosted images sometimes 403 with a referrer). An onError handler swaps to the fallback, so a dead URL degrades identically to a missing one.
  • Fallback: first letter of name, uppercase, font-ui, on a Penny-palette disc — bg-navy text-paper — so initials look designed, not like an error state.

It stays presentation-only: no fetching, no Clerk knowledge, no member types — the shared package must not learn Penny's API shapes.

AvatarStack

Props: { people: Array<{ name: string; imageUrl?: string | null }>; size?: "xs" | "sm" }. Renders the first two entries as overlapping Avatars: a flex row with -ml-2 on the second, each with ring-2 ring-paper so the overlap reads as two distinct discs against the drawer background. Later entries are ignored for now (two-person households are the product reality; a "+N" chip is a five-line addition when a third member exists).

useHouseholdMembers

App-level hook in frontend/src (not the shared package — it knows Penny's API). Fetches GET /api/household/members once via the existing authedFetch/authHeaders helpers, returns { members, me, membersById }, caches in module state for the session. Called in AppShell and passed down as props, mirroring how getToken already travels.

Failure mode: on fetch error it returns an empty list — the sidebar shows no stacks and the transcript renders as today. Avatars are decoration; their absence must never block chat.

Considered and set aside: using Clerk's useUser() for the viewer's own picture. It works, but it would create two avatar sources with two shapes; since the members endpoint already flags is_you, one source serves everything.
