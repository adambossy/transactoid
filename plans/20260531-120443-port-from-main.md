### 2c. Memory / workspace

The current Transactoid memory model is **a directory of markdown files the agent reads and writes** вЂ” not the agent-harness `MemdirLongTermMemory` schema (`MEMORY.md` + `topics/` + `logs/`, hashed-per-project). These are two different abstractions.

**Decision:** port the workspace-as-filesystem model. Don't try to map onto `MemdirLongTermMemory`. Concretely:
- Port `src/transactoid/workspace.py` в†’ `backend/transactoid/workspace.py`. Default `~/.transactoid/`; `$TRANSACTOID_WORKSPACE` override.
- Port `src/transactoid/rules/loader.py` в†’ same path (used by the categorizer to inject merchant rules into the LLM prompt; **not** a tool surface).
- Give the agent read/write/edit on the workspace via agent-harness `FilesystemTools` + an `InProcessSandbox(root=resolve_memory_dir())`. That gives the agent `read`, `write`, `edit`, `ls`, etc. on `~/.transactoid/memory/` вЂ” which is what `edit-merchant-rules-memory` and `generate-budget` need.
- **Skip** `bootstrap/initialization.py` + `memory/index_generation.py` (the auto-index sync) for the MVP. The agent can read/write `index.md` itself if it wants; the auto-generator is polish.

## 7. Risks & flags

- **Localhost Plaid Link** (`localhost:8443`) ports as-is for local dev. Anyone running the new dev backend on a remote machine will find Plaid Link broken. Flag in `backend/README.md`. The proper rearchitecture lives in [[productionize-transactoid]] В§3 (centralized `/plaid/callback`).
- **Global singletons** (knox findings: taxonomy `_category_id_cache`, `_initialized_roots`, MCP server's module-level globals) ride along. They're fine for a single-user dev process; the productionization plan owns the refactor.
- **Categorizer's LLM provider** вЂ” current code uses an LLM client (likely OpenAI) for categorization. We're already on OpenAI in the agent backend; confirm the categorizer module's model name during port and align (e.g. `gpt-5.5` or whatever the YAML prompt specifies).
- **Sync vs async** вЂ” most current tool functions are sync. agent-harness's `@tool` expects async; sync calls will block the event loop. I'll wrap sync calls in `asyncio.to_thread` inside each `@tool` wrapper. Cheap and correct.
- **`run_sql` returns DB rows** вЂ” current shape returns a `dict[str, Any]` with rows. Confirm the JSON-serializability path; some columns (datetimes, Decimals) need `default=str` or a custom encoder in the tool-output serialization. I'll handle in the port.
